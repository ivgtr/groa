## 完了条件
- [x] `packages/generate/src/session-runner.ts` に `runSession()` を実装
- [x] 全モード（tweet / converse / multi / chat）がこの1関数を通る

- [x] コールバック: `onTurnComplete` でターン完了通知、`getUserInput` でchat入力受付
- [x] 各ターンで retrieve → プロンプト構築 → LLM生成 のフローを実行
- [x] tweetモード: maxTurns=1 で既存動作と等価
- [x] converseモード: 前ターンの履歴をコンテキスト転写して蓄積
- [x] multiモード: ラウンドロビンで話者切替、systemプロンプトを話者に切替
- [x] chatモード: getUserInput() → retrieve → 応答生成 のループ
- [x] LlmBackend / Embedder に直接依存（pipeline 層を経由しない）
- [x] Node.js固有APIに依存しない（CLI/Web共通）
- [x] ユニットテスト（モックLLM + モックEmbedder）でtweetモードの動作を検証
