- [x] vitest.config.ts がルートに存在し、--project でパッケージ単位実行が可能
- [x] ESLint設定が存在し、import/no-cycle ルールが有効
- [x] `pnpm build` でルートからの一括ビルドが成功する
- [x] `pnpm test` でルートからの一括テスト実行が成功する（テストが0件でもエラーにならない）
- [x] `pnpm lint` でルートからの一括lint実行が成功する

## 実装メモ
- TypeScript 5.x + pnpm workspace + Vitest + ESLint
- パッケージ間のビルド順序は pnpm の workspace protocol で自動解決
- ルート tsconfig はベース設定のみ（composite/outDir/rootDir は各パッケージ側で宣言）
- web パッケージは DOM/JSX/Bundler モード対応の tsconfig を設定
- ESLint は eslint-plugin-import-x を使用（import/no-cycle 対応）
