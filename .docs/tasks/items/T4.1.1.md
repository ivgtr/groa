# T4.1.1: テキスト正規化・フィルタリング・TweetCorpus生成

- **ステータス**: Done
- **優先度**: Must
- **サイズ**: M
- **対応要件**: R4 / F4.1

## 目的
生ツイートデータをクリーニング・フィルタリングし、後続ステップで使用するTweetCorpusを生成する

## 完了条件
- [x] URLが `[URL]` プレースホルダに置換される
- [x] メンション（@ユーザー名）が除去される
- [x] 連続する空白が単一スペースに正規化され、前後の空白が除去される
- [x] リツイートが除外される
- [x] テキストがURLのみのツイートが除外される
- [x] 正規化後テキストが最小文字数（デフォルト5文字）未満のツイートが除外される
- [x] ボイラープレートパターンに一致するツイートが除外される
- [x] フィルタが合成可能な設計で、新規フィルタの追加が既存コードの変更を要さない
- [x] CorpusMetadata（totalCount, dateRange, filteredCount）が正しく集計される
- [x] 入力0件でVALIDATION_ERROR、前処理後0件で警告+フィルタ緩和案内
- [x] 入力10件未満でエラー停止、100件未満で警告表示して続行
- [x] 50,000件超でエラー
- [x] ユニットテストで各フィルタ条件が個別に検証されている
