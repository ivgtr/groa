# T4.1.1: テキスト正規化・フィルタリング・TweetCorpus生成

- **ステータス**: Todo
- **優先度**: Must
- **サイズ**: M
- **対応要件**: R4 / F4.1

## 目的
生ツイートデータをクリーニング・フィルタリングし、後続ステップで使用するTweetCorpusを生成する

## 完了条件
- [ ] URLが `[URL]` プレースホルダに置換される
- [ ] メンション（@ユーザー名）が除去される
- [ ] 連続する空白が単一スペースに正規化され、前後の空白が除去される
- [ ] リツイートが除外される
- [ ] テキストがURLのみのツイートが除外される
- [ ] 正規化後テキストが最小文字数（デフォルト5文字）未満のツイートが除外される
- [ ] ボイラープレートパターンに一致するツイートが除外される
- [ ] フィルタが合成可能な設計で、新規フィルタの追加が既存コードの変更を要さない
- [ ] CorpusMetadata（totalCount, dateRange, filteredCount）が正しく集計される
- [ ] 入力0件でVALIDATION_ERROR、前処理後0件で警告+フィルタ緩和案内
- [ ] 入力10件未満でエラー停止、100件未満で警告表示して続行
- [ ] 50,000件超でエラー
- [ ] ユニットテストで各フィルタ条件が個別に検証されている

## 依存関係
- 先行: T1.2.1
- 後続: T5.1.1, T6.1.1, T9.1.1

## 実装メモ
- spec.md §4.0, §7.4 に準拠
- フィルタはパイプパターン（合成可能な関数チェーン）で実装
- 処理時間目標: 10,000件で5秒以内
