# T10.1.1: Cosine similarity検索・多様性フィルタリング

- **ステータス**: Todo
- **優先度**: Must
- **サイズ**: M
- **対応要件**: R10 / F10.1

## 目的
トピックに関連し、かつ態度の多様性を確保したツイートを検索する

## 完了条件
- [ ] トピック文字列のEmbeddingが生成される（multilingual-e5-small、ローカル）
- [ ] Phase 1: Cosine similarity で上位 topK*6 件の候補が取得される
- [ ] Phase 2: sentiment/category の多様性を確保しつつ 2*topK 件が選定される
- [ ] 結果が前半（生成用）と後半（評価用）に分割される
- [ ] topK, sentimentDiversity, categoryDiversity の設定パラメータが反映される
- [ ] 候補不足時のフォールバック（取得可能な全件から処理、件数不足時は半分に分割）が動作する
- [ ] ユニットテストで検索・フィルタリング・分割が検証されている

## 依存関係
- 先行: T9.1.2, T1.2.2
- 後続: T11.1.1, T12.1.1, T13.1.2

## 実装メモ
- spec.md §5.1 に準拠
- Cosine similarity はベクトルの内積 / (ノルムの積) で計算
- 多様性フィルタはグリーディ選択（最もスコアが高い候補から、多様性条件を満たすものを順次追加）
