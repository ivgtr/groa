# T17.3.1: groa init モデル指定オプション追加

- **ステータス**: Done
- **優先度**: Must
- **サイズ**: S

## 目的
`groa init` 実行時にモデルIDを指定できるようにし、openrouter バックエンド選択時の APIキー プレースホルダを設定する。

## 完了条件
- [x] `groa init --models.haiku <id>` でモデルIDが `groa.json` の `models.haiku` に書き込まれる
- [x] `groa init --models.sonnet <id>` で同様に `models.sonnet` に書き込まれる
- [x] `groa init --models.opus <id>` で同様に `models.opus` に書き込まれる
- [x] `groa init --backend openrouter` 選択時に `apiKeys.openrouter` が `"${OPENROUTER_API_KEY}"` に設定される
- [x] `groa init --backend anthropic` 選択時に `apiKeys.anthropic` が `"${ANTHROPIC_API_KEY}"` に設定される（既存動作の維持）
- [x] モデルオプション未指定時は `models.haiku` / `sonnet` / `opus` が `null` のまま出力される
- [x] ユニットテストで各オプションの動作が検証されている
- [x] 型エラーが0件
- [x] テストが通る
