# T9.1.2: バッチEmbedding生成・インデックス永続化

- **ステータス**: Done
- **優先度**: Must
- **サイズ**: M
- **対応要件**: R9 / F9.1

## 目的
全ツイートのEmbeddingをバッチ生成し、再利用可能な形式で永続化する

## 完了条件
- [x] TweetCorpus の全ツイートに対して384次元のEmbeddingが生成される
- [x] Float32Array でベクトルが保持される
- [x] CLI: JSONファイルとして永続化される
- [x] Web: IndexedDBに保存される（T15.1.3で対応）
- [x] 再実行時、既存のEmbeddingIndexが有効ならスキップされる
- [x] サイズ見積: 8,000件で約12MB（Float32Array）、JSON約18-25MB
- [x] 処理時間が10,000件で2分以内
- [x] ユニットテストでEmbedding生成と永続化が検証されている

## 依存関係
- 先行: T9.1.1, T4.1.1
- 後続: T10.1.1, T13.1.1

## 実装メモ
- spec.md §4.5 に準拠
- バッチサイズは環境のメモリに応じて動的調整
- EmbeddingIndex にはモデルIDを記録
- JSON永続化は `serializeEmbeddingIndex` / `deserializeEmbeddingIndex` で Float32Array ↔ number[] 変換
- スキップロジックはパイプライン層（StepCacheManager）が担当
- Web IndexedDB はT15.1.3で対応（embed パッケージのシリアライズ関数をそのまま利用可能）
