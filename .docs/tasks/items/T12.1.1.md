# T12.1.1: 評価プロンプト構築・スコア算出・合格判定

- **ステータス**: Done
- **優先度**: Must
- **サイズ**: M
- **対応要件**: R12 / F12.1

## 目的
生成テキストが「その人が書いたように読めるか」を元ツイートとの直接比較で評価する

## 完了条件
- [x] 元ツイート（評価用5-10件）と生成テキストを並べた評価プロンプトが構築される
- [x] ボイスバンクから5件が評価コンテキストに含まれる
- [x] PersonaDocumentを介した間接評価は行わない（循環性回避）
- [x] authenticity, styleNaturalness, attitudeConsistency（各0.0-10.0）が算出される
- [x] rationale（評価根拠の自然言語）が出力される
- [x] Sonnet, temperature 0.0 で評価される
- [x] authenticity >= 6.0（設定で変更可能）で合格判定される
- [x] evaluation フィールドが GeneratedText に付与される（一度付与されたら不変）
- [x] バリデーション失敗時は最大2回リトライ、全失敗でエラー停止
- [x] ユニットテスト（モック）で評価プロンプト構築とスコア出力が検証されている
